Detailed Description

Abstract representation of a game world.

Used to interact with ECS layers and runtime data.

GameWorld serves two primary purposes here:

  1. to centralize entry points for common ECS operations and ensure correct coordination across modules.
  2. to theoretically allow replacement with a more complete, more robust ECS (e.g., flecs) at some later stage, if desired.
